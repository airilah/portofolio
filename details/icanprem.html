Portfolio Description for Website Application I Can Premium

1. Project Overview

I Can Premium is a web-based platform developed as a personal project using Laravel 10, PHP 8.3, Bootstrap, and JavaScript. The primary goal of this system is to support the business of selling premium applications and account subscriptions through a structured, secure, and user-friendly platform.

This system is designed to:

  • Provide clear product information (description, price, category, stock status).
  • Facilitate transactions between buyers and sellers.
  • Provide a shopping cart.
  • Integrate payment features.
  • Provide an admin dashboard to manage products, orders, and customer data.

2. Planning and Requirements Analysis

a. Problem Identification
  • Premium account sales were previously handled manually through chat, causing transaction records to be inconsistent.
  • Buyers often had difficulty comparing products because information was scattered across multiple places.
  • Payment confirmation and account delivery processes usually took time.
b. System Objectives
  • To build a centralized platform for displaying premium product information.
  • To simplify the purchase process, allowing users to browse, add to cart, and checkout easily.
  • To help sellers manage products, stock, order transactions, and payments efficiently.
c. Functional Requirements
  • User registration and login.
  • Product catalog page.
  • Product detail page (description, price, benefits).
  • Shopping cart (add, update, delete items).
  • Checkout process and input of order information.
  • Payment confirmation (upload proof or via gateway).
  • Order history page for customers.
  • Admin dashboard for managing products, orders, and users.
d. Non-Functional Requirements
  • Responsive interface on multiple devices using Bootstrap.
  • Fast performance and lightweight interface.
  • Basic security such as CSRF protection, validation, and authentication using Laravel.
  • Clean and scalable code structure.

3. System Design

a. Database Design

Uses relational database (e.g., MySQL) with tables such as:

  • users — stores user information (buyers and admin).
  • products — stores premium product details.
  • categories — stores product categories (optional).
  • orders — stores order records.
  • order_items — stores items for each order.
  • payments — stores payment transactions and status.
b. System Architecture

Follows MVC (Model-View-Controller) architecture of Laravel:

  • Models: User, Product, Order, OrderItem, etc.
  • Controllers:
    • ProductController — handles product display.
    • CartController — handles shopping cart logic.
    • OrderController — handles checkout process.
    • Admin\ProductController, Admin\OrderController — admin-side controllers.
    • etc
  • Views: created using Blade templates combined with Bootstrap layout.
c. UI/UX Design
  • Homepage showcasing featured products or promo items.
  • Navigation menu: Home, Product Catalog, Cart, Order History, Login/Register.
  • Admin interface focused on clarity and usability.

4. Development and Implementation

a. User Authentication
  • Uses Laravel authentication system (e.g., Breeze/Fortify).
  • Supports register, login, logout, and restricted access.
b. Product Management
  • CRUD features for premium products.
  • Allows updating product prices, descriptions, stock, and availability.
c. Shopping Cart
  • User can add products into the cart and modify quantity.
  • Cart summary includes total calculation.
d. Checkout and Order Processing
  • System stores order data into orders and order_items table.
  • Order default status set to pending.
e. Payment Feature
  • Initially supports manual payment confirmation by uploading proof.
  • Can later be integrated with payment gateways.
f. Admin Dashboard
  • Admin can view incoming orders and change their status (pending, paid, delivered).
  • Admin can deliver account credentials after order verification.

5. Testing Phase

a. Functional Testing
  • Test user flow: register → login → browse → add to cart → checkout.
  • Verify order data stored correctly in database.
  • Test admin access to product and order management.
b. Validation Testing
  • Test invalid input submission to ensure proper error messages.
c. UI Testing
  • Check responsiveness on mobile, tablet, and desktop.
d. Security Testing
  • Ensure admin pages cannot be accessed without login.
  • CSRF protection working properly across forms.

6. Deployment Stage

a. Server Preparation
  • Deploy to hosting/VPS supporting PHP 8.3 and MySQL.
  • Upload project files using Git or FTP.
b. Server Configuration
  • Configure .env (database, APP_URL, email settings, etc.).
  • Run deployment commands
  • Map document root to public folder.
c. Optimization
  • Run performance commands
  • Re-test full workflow on live deployment.

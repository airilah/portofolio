Portfolio Description for Website PPS Jokotole Diklat Kodim 0829

1. Project Overview

The PPS Jokotole Diklat Kodim 0829 website is a web-based platform developed as a team project to support the administration, promotion, and profile information of the traditional martial arts school PPS Jokotole.

The goal of this website is to provide a modern, accessible, and responsive digital system that presents:

  • The organization profile
  • Training programs and locations
  • Students’ achievements
  • Official merchandise
  • Online registration for new students

The website is built using:

  • Frontend: HTML5, CSS3, Bootstrap, JavaScript
  • Backend: PHP
  • Database: MySQL
  • Version Control: Git & GitHub
  • Deployment: Shared hosting server

2. Project Objectives

a. For the Organization (Admins & Coaches)
  • Digitize new student registration through an online form.
  • Provide a platform to present the history and identity of the school.
  • Showcase athletes’ achievements in a structured way.
  • Support the branding of PPS Jokotole with a professional online presence.
b. For the Community (Users & Parents)
  • Make it easier to register new students without coming directly to the dojo.
  • Access information about the school’s background, vision, and mission.
  • View a gallery of achievements and events.
  • See official merchandise and contact information.

3. System Planning and Analysis

a. Problem Identification

Before the website was built, the school faced several issues:

  • New student registration was done manually using paper forms.
  • Official information about the school was not centralized and not easily accessible.
  • Documentation of students’ achievements was not well organized.
  • The organization did not yet have a strong digital presence for promotion.
b. User Needs Analysis
  • Prospective students & parents: need simple registration and clear information.
  • Admins: need automatic data storage and easier management of student data.
  • Coaches: need a place to show training activities and achievements.
c. Functional Requirements
  • Online registration form for new students.
  • School profile page (history, vision, mission, logo meaning).
  • Achievements gallery (photos, titles, levels, names).
  • Merchandise section.
  • Maps/location and contact information.
d. Non-Functional Requirements
  • Responsive design (mobile-friendly).
  • Simple and intuitive user interface.
  • Secure and structured database.
  • Visual design that matches the martial arts theme.

4. System Design

a. UI/UX Design

The team designed the layout using simple wireframes and Bootstrap-based grids. The key elements include:

  • Top navigation bar with menu items: Home, About Us, Maps, Merchandise, Achievement Gallery.
  • Main hero section with a background image and “Register Now” button.
  • Profile section with logo, history, vision & mission, and symbol explanation.
  • Modal-based registration form for new students.
  • Card-based layout for displaying achievements and merchandise.
b. Database Design

Several main tables are used:

  • students – stores new student registration data.
  • achievements – stores competition results and awards.
  • merchandise – stores merchandise items and descriptions.
  • admins – stores administrator login credentials.

5. Development Process

a. Frontend Development

The frontend team was responsible for:

  • Creating the landing page with a hero section and background image of the team.
  • Building the profile page with collapsible sections for history, vision & mission, and logo meaning.
  • Designing a modal popup form called “Form Pendaftaran Murid Baru” (New Student Registration Form).
  • Designing cards to display achievements and merchandise items.
  • Ensuring all pages are responsive using Bootstrap.
b. Backend Development

The backend team focused on:

  • Implementing PHP scripts to process and store registration data into MySQL.
  • Validating inputs, including file uploads for formal photos.
  • Creating basic admin pages to view registered students and manage data.
  • Connecting the frontend forms and pages with database operations.
c. Content and Documentation

A separate part of the team handled:

  • Collecting historical information about PPS Jokotole.
  • Writing content for the “About Us” and profile sections.
  • Gathering pictures and details of students’ achievements.
  • Preparing descriptions for merchandise and activities.

6. Testing Phase

a. Functional Testing
  • Testing the entire registration flow, from filling the form to saving data in the database.
  • Ensuring uploaded photos are correctly stored and validated.
  • Verifying that achievement and merchandise data are displayed correctly on the website.
b. UI Testing
  • Checking layout on mobile phones, tablets, and desktop screens.
  • Ensuring text, buttons, and images are properly aligned and readable.
  • Testing modal dialogs and navigation menus.
c. UX Testing
  • Evaluating how easily users can find the registration form.
  • Checking whether the information flow (Home → About Us → Registration) feels natural.
  • Gathering informal feedback from test users and improving button labels and instructions.

7. Deployment Stage

a. Preparation
  • Bundling all project files (frontend, backend, and assets) into a deployable structure.
  • Exporting the MySQL database from the development environment.
b. Upload to Hosting
  • Uploading the website files to a shared hosting server via FTP or a control panel.
  • Importing the database using phpMyAdmin on the hosting server.
  • Updating database connection settings (host, username, password, database name) in the PHP configuration file.
c. Post-Deployment Testing
  • Re-testing the registration form on the live domain.
  • Checking all navigation links and pages.
  • Ensuring that data is correctly stored and retrieved using the live database.

8. Benefits and Impact

a. For PPS Jokotole Organization
  • More professional public image with an official website.
  • Structured and digital student registration system.
  • Better documentation of history, values, and achievements.
  • A centralized platform for promotion and communication.
b. For the Community and Parents
  • Easy online registration from anywhere and at any time.
  • Transparent information about the school and its achievements.
  • Increased trust through a formal and informative website.

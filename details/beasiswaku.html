Portfolio Description for Website Application PKP-RI Bangkalan

1. Project Overview

The PKP-RI Bangkalan hall reservation website is a web-based platform developed as a team project using Laravel 10, PHP 8.3, Bootstrap, and JavaScript.

This application is designed to support the business process of hall rental at PKP-RI Bangkalan for various events, such as seminars, organizational meetings, family gatherings, training, and social activities. The system provides an online booking platform that can be accessed by both customers and administrators.

2. Project Goals

a. Problems in the Previous Process
  • Reservations were handled manually via phone or in-person visits.
  • Hall availability schedules were not clearly visible to customers.
  • Renter data and reservations were recorded manually and prone to errors.
  • Administrators had to manually manage bookings, which was time-consuming.
b. Objectives of the System
  • Provide an online booking platform that is accessible anytime and anywhere.
  • Display hall availability schedules more clearly and transparently.
  • Help PKP-RI staff manage customer booking data in a structured way.
  • Automatically store booking and transaction history in a database.

3. Requirements Analysis

a. Functional Requirements
For Customers
  • View hall information, facilities, and rental prices.
  • View the availability schedule for each hall.
  • Create an account or log in before making a reservation.
  • Make reservations by selecting date, time, and type of event.
  • Submit payment or upload payment proof.
  • View the status of their reservations.
For Administrators
  • Manage hall data (name, capacity, facilities, price, and status).
  • View and manage incoming bookings.
  • Update booking status (pending, approved, rejected, completed).
  • Manage customer data when needed.
  • View booking and transaction reports.
b. Non-Functional Requirements
  • Responsive design that works well on mobile and desktop devices (using Bootstrap).
  • Secure authentication and authorization using Laravel's features.
  • Stable performance and efficient database access.
  • Scalable structure to support future feature additions, such as payment gateway integration.

4. System Design

a. Database Design

The system uses a relational database (such as MySQL) with several main tables:

  • users — stores customer and admin account data.
  • buildings or halls — stores hall data (name, capacity, facilities, price).
  • bookings — stores booking records made by customers.
  • payments — stores payment information and uploaded payment proof.
  • time_slots — stores detailed booking time frames (if used).
  • etc

Relationships between tables are defined using foreign keys to maintain data integrity and consistency.

b. Architecture Design

The application follows the MVC (Model-View-Controller) architecture using Laravel:

  • Models: User, Hall, Booking, Payment, etc.
  • Controllers:
    • BookingController — handles booking logic and workflows.
    • HallController — manages hall data and details.
    • AdminController — manages admin dashboard, reports, and approvals.
    • etc
  • Views: created using Blade templates combined with Bootstrap components.
c. UI/UX Design
  • Designing a homepage that introduces PKP-RI Bangkalan and the available halls.
  • Providing a clear navigation menu: Home, Halls, Schedule, Booking, Login/Register.
  • Designing a booking form that is simple and easy to understand.
  • Designing an admin interface for viewing tables of bookings and managing their status.

5. Implementation Phase

a. Backend Implementation
  • Setting up routes for customer and admin roles.
  • Using middleware to protect admin-only pages.
  • Using Laravel migrations and Eloquent ORM for database operations.
  • Implementing input validation to ensure correct and safe data.
  • Implementing booking logic, including checking hall availability based on selected date and time.
b. Frontend Implementation
  • Building page layouts using Blade templates and Bootstrap grid system.
  • Ensuring responsive design across different screen sizes.
  • Using JavaScript for interactive elements such as:
    • Date picker or calendar for selecting booking dates.
    • Dynamic display of booking status and notifications.
    • Additional client-side validation.
c. Team Collaboration
  • Work was divided among team members: backend, frontend, database, and testing.
  • Using Git and a shared repository to manage source code and collaborate effectively.
  • Regular discussions to align features, handle bugs, and improve UI/UX.

6. Testing Phase

a. Unit and Functional Testing
  • Testing booking creation with valid and invalid data.
  • Testing date and time validation to avoid double-booking.
  • Testing login, logout, registration, and role-based access.
b. End-to-End User Flow Testing
  • Customer scenario: register → login → select hall → choose date and time → submit booking → upload payment proof → check booking status.
  • Admin scenario: login → view incoming bookings → verify payment → approve or reject booking → update booking status.
c. Usability Testing
  • Testing the system with several trial users to evaluate ease of use.
  • Improving date format display and booking status indicators based on feedback.

7. Deployment Phase

a. Server and Hosting Preparation
  • Choosing hosting or a VPS that supports PHP 8.3 and MySQL.
  • Configuring a domain or subdomain for the PKP-RI hall reservation system.
b. Deployment Steps
  • Uploading the Laravel project files via Git or FTP.
  • Setting up the .env configuration file for the production environment (database, APP_URL, etc.).
  • Running the following commands on the server
  • Pointing the web server's document root to the public directory of the Laravel project.
c. Optimization and Final Testing
  • Running optimization commands
  • Retesting key user flows on the live server:
    • Customer registration and login.
    • Hall browsing and booking creation.
    • Payment proof upload and status checking.
    • Admin verification and approval process.

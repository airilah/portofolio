Portfolio Description for Website Klasifikasi Kanker Kulit dengan VGG19

1. Project Overview

This website is a web-based application for skin cancer classification from dermoscopic images. Users can upload a skin image in .jpg / .jpeg format, and the system will process the image using a VGG19 deep learning model to predict the class of the lesion.

The application displays:

  • The predicted class (for example: melanoma, melanocytic nevi, etc.).
  • The confidence score (prediction probability in percent).
  • A probability chart for each class.
  • A research summary and the main program/code used in the project.

This is a personal project developed using:

  • Python
  • TensorFlow / Keras (VGG19)
  • NumPy, Pandas, Matplotlib, Seaborn for data processing and visualization
  • Streamlit (or a similar Python web framework) for the web interface

2. Background and Goals

a. Background

Skin cancer, especially melanoma, is one of the fastest growing cancers worldwide. Early detection is extremely important, because if the cancer is detected early, the chances of survival can increase significantly.

However, diagnosing skin cancer requires expertise and access to proper medical tools, which are not always available in every region. Therefore, computer-aided diagnosis using image classification models can support medical staff and also be used as an educational tool.

b. Project Goals
  • To implement an image classification model using the VGG19 architecture to distinguish between different types of skin lesions.
  • To apply Random Over Sampling (ROS) to handle class imbalance in the dataset.
  • To present the research results in the form of an interactive website that:
    • Can be used for academic demonstration and education.
    • Displays a research summary, research program/code, and a live image prediction feature in one application.

3. Data Preparation Phase

a. Dataset

The project uses the HAM10000 dataset (Human Against Machine with 10000 training images), which contains several types of skin lesions, such as:

  • Actinic keratoses
  • Basal cell carcinoma
  • Benign keratosis-like lesions
  • Dermatofibroma
  • Melanocytic nevi
  • Melanoma
  • Vascular lesions
b. Image Preprocessing
  • Resize all images to match the VGG19 input size (for example, 224 x 224 pixels).
  • Normalize pixel values (for example, scaling to the range 0â€“1).
  • Split the dataset into:
    • Training set
    • Validation set
    • Test set
C. Handling Class Imbalance

Because the number of images per class is not balanced, the project applies: Random Over Sampling (ROS) to duplicate samples from minority classes until the class distribution becomes more balanced.

This helps the model avoid bias towards majority classes and improves its ability to recognize minority classes such as melanoma.

4. VGG19 Model Design

a. Transfer Learning with VGG19
  • Import the VGG19 model from tensorflow.keras.applications.vgg19.
  • Use ImageNet pretrained weights to leverage previously learned visual features.
  • The convolutional base of VGG19 is often frozen (not trainable) in the initial training phase to retain general image features such as edges and textures.
b. Architecture Adaptation
  • Add a Global Average Pooling or Flatten layer after the VGG19 feature extractor.
  • Add several Dense layers with ReLU activation and Dropout to reduce overfitting.
  • Use a Softmax output layer with the number of neurons equal to the number of classes in the dataset.
c. Training Configuration
  • Loss function: Categorical Crossentropy.
  • Optimizer: for example, Adam.
  • Metrics: Accuracy, and optionally Precision, Recall, AUC.
  • Callbacks such as:
    • ModelCheckpoint — to save the best performing model.
    • EarlyStopping — to stop training when validation performance stops improving.

5. Training and Evaluation

a. Training
  • The model is trained on the training set (after applying ROS) for several epochs.
  • During training, training and validation loss and accuracy are monitored to check whether the model is learning properly.
  • The best model is saved to a file, for example: model_vgg19_skin_cancer.h5.
b. Evaluation
  • The final model is evaluated using the test set, which was not used during training.
  • Metrics such as accuracy, confusion matrix, and classification report (precision, recall, F1-score) are calculated.
  • From the evaluation, we can see how well the model distinguishes between melanoma and non-melanoma lesions and other classes.

6. Website / Web Application Development

The website interface includes several main menus, as illustrated by the screenshots:

a. Research Summary
  • Displays the research title.
  • Presents background, problem statement, and proposed solution.
  • Explains the use of Random Over Sampling (ROS) and VGG19 as the main methods.
  • This page is written in a structure similar to an introduction chapter in a thesis or journal, but shown directly in the web page.
b. Research Program
  • Displays the main Python code used in the research.
  • Includes:
    • Library imports (NumPy, Pandas, Matplotlib, Seaborn, TensorFlow, Keras, etc.).
    • Data preprocessing, model creation, training, and evaluation sections.
  • This page serves for documentation and transparency, especially for academic or research purposes.
c. Image Prediction
  • Provides a file upload component where users can drag and drop or browse a skin image.
  • The uploaded image is displayed on the page as a preview.
  • After processing, the system shows:
    • Prediction Result: the predicted class (for example: melanoma).
    • Confidence Level: e.g. 79.31%.
    • Probability for Each Class: a bar chart showing probabilities for all classes so that the user can compare them visually.

7. Model Integration with the Website

a. Loading the Model
  • The trained VGG19 model is loaded when the web application starts.
  • This avoids reloading the model every time a user makes a prediction, improving performance.
b. Prediction Function
  • When a user uploads an image:
    • The image is read using PIL or OpenCV.
    • The image is resized to the model input size.
    • Pixel values are normalized and converted into a NumPy array with batch dimension.
    • The image is passed to model.predict() to obtain a probability vector.
  • The index of the highest probability is taken as the predicted class.
  • The corresponding probability is converted into a percentage as the confidence score.
c. Visualization
  • The prediction result and confidence are shown as text, along with the original uploaded image at the top of the page.
  • A bar chart of the class probabilities is created using Matplotlib or Plotly and rendered in the web page to help users interpret the model output.
